## 专题一：香港政府如何防止受雇人员逃税

香港税务局（Inland Revenue Department, IRD）构建了一套以雇主申报为核心、多维度信息交叉验证为辅的薪俸税合规管理体系。与内地个税的全员全额扣缴申报与预扣预缴制度不同，香港更侧重于雇主的年度申报、离职/离港等关键节点的报告义务，并辅以严厉的处罚机制来确保税收合规。

### 1. 雇主强制申报义务：IR56系列表格解析

雇主在薪俸税管理中扮演着“前线把关人”的角色，其核心职责是通过填报IR56系列表格，向税务局提供雇员的薪酬信息。[[63]](https://www.ird.gov.hk/eng/tax/ere_obl.htm)

#### 表格适用场景与填报时点

- **IR56B (雇主填报的薪酬及退休金报税表)**: 这是最核心的年度申报表格。雇主需在每个课税年度结束后的一个月内（即每年4月份），为所有在职员工（无论收入高低）填报上一年度（从4月1日到次年3月31日）的全部薪酬福利信息。[[64]](https://www.ird.gov.hk/eng/tax/ere.htm) BIR56A是雇主收到的申报通知书封面，IR56B则是每个员工的具体收入明细。
    
- **IR56E (新聘雇员通知书)**: 当企业雇佣一位新员工时，必须在员工入职后的**三个月内**提交IR56E表格。[[71]](https://www.activpayroll.com/global-insights/hong-kong) 这使得税务局能及早掌握新雇员的信息，建立税务档案。
    
- **IR56F (终止受雇通知书)**: 当一名员工离职，但**仍会留在香港**时，雇主需在其离职前一个月提交IR56F表格。[[65]](https://sleek.com/hk/resources/ir56-forms/) 这确保了员工在更换工作期间的税务记录不会中断。如果某雇员的IR56F已提交，则年度申报时无需再为其提交IR56B。[[64]](https://www.ird.gov.hk/eng/tax/ere.htm)
    
- **IR56G (雇员行将离港通知书)**: 这是监管力度最强的表格。当员工计划**离开香港超过一个月**（无论是永久离开还是长期外派），雇主必须至少提前一个月提交IR56G表格，并有法定义务暂扣应付给该员工的所有款项，直至收到税务局发出的“同意释款书”（Letter of Release）。[[65]](https://sleek.com/hk/resources/ir56-forms/)[[68]](https://osome.com/hk/blog/how-to-file-employers-return-form-bir56a-ir56b/)
    
- **IR56M (支付酬金给非雇员的通知书)**: 如果企业向非雇员的本地个人（如顾问、代理人、特约艺术家等）支付酬金，且在某一年度内支付总额超过特定金额，则需填报此表。
    

**提示** IR56系列表格构成了雇主申报义务的完整闭环，覆盖了员工从入职、在职、离职到离港的全生命周期。雇主必须严格遵守各项表格的提交时限和场景要求，避免遗漏。

|   |   |   |   |
|---|---|---|---|
|表格类型|适用场景|提交时限|关键要点|
|**IR56B**|年度申报所有在职雇员的薪酬|每年4月（课税年度结束后1个月内）|全面、准确反映所有应税收入|
|**IR56E**|新雇佣员工|入职后3个月内|建立新雇员税务档案|
|**IR56F**|员工离职，但仍留港|离职前1个月|确保税务记录连续性|
|**IR56G**|员工将离港超过1个月|预计离港日前至少1个月|**须暂扣款项**，等待“放行信”|
|**IR56M**|向非雇员的个人支付酬金|年度申报|针对顾问、自由职业者等|

#### 合规要点清单

- **全面性**：申报的收入必须完整，包括工资、津贴、奖金、佣金、假期薪酬、约满酬金，以及各种非货币性福利的等值现金价值（如住房福利、股票期权收益等）。[[68]](https://osome.com/hk/blog/how-to-file-employers-return-form-bir56a-ir56b/)
    
- **准确性**：所有数据的填报都应基于真实的工资单和雇佣合同记录。
    
- **及时性**：严格遵守每个表格的提交截止日期，特别是离职和离港的提前通知义务。
    

#### 常见误区

- **误区一：只为高收入员工申报IR56B**。错误。无论员工收入是否达到纳税门槛，只要存在雇佣关系，就必须为其提交年度IR56B。
    
- **误区二：员工离职后就与公司无关**。错误。雇主有法定责任在员工离职（IR56F）或离港（IR56G）前通知税务局。
    
- **误区三：忘记申报非香港子公司支付的薪酬**。如果香港公司的雇员因向集团内非香港子公司提供服务而获得报酬，香港雇主仍需在IR56B中合并申报该笔收入。[[64]](https://www.ird.gov.hk/eng/tax/ere.htm)
    

#### 企业动作建议

- **建立台账**：HR部门应建立清晰的员工入职、异动、离职台账，并设置日历提醒，确保各项IR56表格的按时提交。
    
- **薪酬数据接口**：财务或薪酬系统应能便捷地导出符合IR56B格式要求的年度薪酬数据，减少手动汇总错误。
    
- **离职/离港流程整合**：将IR56F/IR56G的提交内嵌到标准的员工离职或外派流程中，作为强制性步骤。
    

### 2. 源头扣缴机制：与内地的制度性差异

香港的薪俸税管理与中国内地在源头扣缴机制上存在根本性差异，理解这一点对跨境企业的合规至关重要。

#### 香港：无全面的“即赚即缴”（PAYE）制度

香港**没有**类似内地或其他国家的全面薪酬预扣（Pay-As-You-Earn, PAYE）制度。[[71]](https://www.activpayroll.com/global-insights/hong-kong) 雇主在日常发放工资时，**通常无需按月代扣代缴雇员的薪俸税**。雇员的税务责任是独立的，他们需要根据税务局发出的评税通知书，自行在指定期限内（通常分两期缴纳）向政府缴税。[[59]](http://m.toutiao.com/group/7501564154651787810/)

唯一的例外是**离港清税**。

#### “离港清税”下的暂扣义务 (IR56G)

当雇主为即将离港的员工提交IR56G表格后，法律强制要求雇主**暂扣**自提交日起应支付给该员工的所有款项，包括薪金、佣金、奖金、花红及其他任何形式的报酬（money or money's worth），不得支付。[[12]](https://carbonik.com/en-hk/hr-guide/hong-kong/termination)[[68]](https://osome.com/hk/blog/how-to-file-employers-return-form-bir56a-ir56b/) 此暂扣义务会一直持续，直到雇主收到税务局发出的“**同意释款书**”（Letter of Release）为止。这份文件证明该员工已结清所有在港税款。

**风险** 如果雇主在未收到“放行信”的情况下，擅自将款项支付给离港员工，该雇主将可能需要承担该员工所欠的全部税款。

#### 与内地个税预扣预缴的对比

|   |   |   |
|---|---|---|
|对比维度|**香港薪俸税**|**中国内地个人所得税**|
|**申报主体**|雇主申报信息，雇员自行缴税|企业（扣缴义务人）是申报与缴纳的双重主体|
|**源头扣缴义务**|**一般无**（PAYE非强制）。仅在员工离港（IR56G）时有暂扣义务|**全面强制**。企业必须按月（或按次）为居民及非居民个人预扣预缴或代扣代缴个税|
|**清算周期**|按课税年度（每年4月1日至次年3月31日）进行一次性评税和缴纳|按月预缴，次年进行年度汇算清缴（居民个人综合所得）|
|**离职/离境处理**|- **离职留港 (IR56F)**：仅申报，无扣缴义务<br>    <br>- **离港 (IR56G)**：提前1个月申报，并暂扣所有未付款项直至税局“放行”|- **离职**：当月正常预扣，无需特殊申报<br>    <br>- **非居民离境**：在离境前办理税款清算|

#### 合规要点清单

- 明确香港“无PAYE”的基本事实，不要错误地按月为员工代扣薪俸税。
    
- 严格执行IR56G场景下的款项暂扣义务，规避连带责任风险。
    
- 向新入职（尤其是有内地背景）的员工清晰解释香港的个人报税缴税制度，避免误解。
    

#### 常见误区

- **误区一：认为香港和内地一样，公司代扣了税就完事了**。错误。香港雇员需独立承担报税和缴税责任。
    
- **误区二：对IR56G的暂扣义务理解为“扣税”**。不完全准确。雇主是“暂扣”（withhold）所有应付款项，而非直接计算并“代缴”税款。具体税额由税务局与员工本人清算。
    
- **误区三：口头答应员工先发钱，等拿到“放行信”再说**。这是严重的违规行为，雇主将直接暴露在替员工缴税的风险之下。
    

#### 企业动作建议

- **员工入职培训**：在员工手册或入职培训中加入香港薪俸税基本制度介绍，特别是与内地的差异对比。
    
- **财务流程设定**：财务部门应明确，除IR56G情况外，不得从员工薪酬中扣除薪俸税。
    
- **离港流程检查**：在处理离港员工的最终薪酬结算时，必须设置强制校验点，确认是否已收到税务局的“同意释款书”。
    

### 3. 数据交叉稽核：多维度的风险监控

税务局并非仅依赖雇主的单一申报来监管。它通过与多个政府及半官方机构的数据交换，构建了一个立体的监控网络，以识别异常和逃税风险。

#### 主要数据交互方与风控逻辑

- **强制性公积金（MPF）受托人**：雇主和雇员的MPF供款额是基于“有关入息”计算的。税务局可将雇主申报的薪酬（IR56B）与MPF受托人记录的供款基数进行比对。若两者出现重大、无法解释的差异，可能意味着薪酬申报不实。
    
- **入境事务处**：对于需要按在港工作天数分摊收入的跨境雇员，其出入境记录是计算在港天数的核心依据。税务局可从入境处获取权威的出入境数据，用以核实纳税人自行申报或雇主报告的留港天数是否准确。
    
- **土地注册处/差饷物业估价署**：雇员享受的“由雇主提供居所”福利是一项重要的应税收入。税务局可以通过土地注册处或差饷物业估价署的数据，核实物业的租值（应评税值），与雇主申报的房屋福利金额进行比对，检查是否存在低报。
    
- **公司注册处**：董事袍金或分红等收入，可通过公司注册处的董事信息与公司年报进行关联分析。
    
- **金融机构（CRS/AEOI框架下）**：下文详述。
    

**被动暴露场景**

- **收入与MPF供款基数不匹配**：HR为降低MPF成本而低报“有关入息”，但财务在IR56B上如实申报了全部薪酬，导致数据矛盾。
    
- **出入境记录与报税天数冲突**：员工为享受“60天豁免”或减少在港计税收入，自行调整了留港天数，但与入境处官方记录不符。
    
- **申报的租金福利远低于市场水平**：公司为高管提供的豪宅，在IR56B中仅象征性地申报了极低的租金价值，与差饷物业估价署的评估租值严重背离。
    

#### 合规要点清单

- 确保向不同机构提交的基数（如薪俸税的应税收入与MPF的有关入息）保持一致性或存在可合理解释的差异。
    
- 教育员工，特别是跨境工作的员工，必须依据官方出入境记录来计算和申报在港/离港天数。
    
- 对提供的非货币福利（如住房），应基于公允价值或官方估值进行合理申报。
    

#### 常见误区

- **误区一：认为税务局只看IR56表格**。错误。税务局拥有多渠道数据，交叉验证是其核心风控手段。
    
- **误区二：随意调整MPF供款基数以“节约成本”**。这种操作会直接在税务局和MPF受托人之间留下数据缺口，是高风险行为。
    
- **误区三：认为员工自己报税，公司就没有交叉验证的风险**。错误。公司提交的IR56B是税务局进行数据比对的基础，若公司申报数据本身与其他来源冲突，税务局同样会向公司发起质询。
    

#### 企业动作建议

- **数据同源**：HR、财务部门应确保用于计算MPF供款和申报薪俸税的薪酬数据口径一致，任何差异都应有清晰的法律依据（例如，某些收入不属于MPF的“有关入息”范畴）。
    
- **留存证据**：对于跨境员工，保留其雇佣合同中关于工作地点的条款、出差记录、费用报销单等，以支持收入分摊的合理性。
    
- **福利公允估值**：对于住房、公司用车等实物福利，应获取并保留市场租金协议、第三方评估报告等作为估值依据。
    

### 4. 离港清税（IR56G）：最严格的监管环节

离港清税是香港税务体系中程序最为严谨、雇主责任最重的环节之一，旨在确保任何即将离开香港的纳税人在离境前结清所有税款。

#### 要求与时间线

- **触发条件**：任何雇员（无论国籍或职位）计划离开香港**超过1个月**。[[65]](https://sleek.com/hk/resources/ir56-forms/)
    
- **雇主责任**：必须在雇员预计离港日的**至少1个月前**，向税务局提交一式两份的IR56G表格。[[9]](https://www.ird.gov.hk/eng/pdf/pam46e.pdf)
    
- **暂扣款项**：从提交IR56G之日起，雇主**必须扣起**所有即将支付给该员工的款项，不得支付，直到收到税务局发出的“同意释款书”（Letter of Release）。[[12]](https://carbonik.com/en-hk/hr-guide/hong-kong/termination)[[14]](https://www.workstem.com/hk/en/glossary/ir56g/)
    
- **员工责任**：员工在收到雇主转交的IR56G副本后，需主动联系税务局，完成最终的税务评定和缴款。
    

#### 暂扣款项的范围

法律规定雇主需暂扣的款项是“**money or money’s worth**”（金钱或金钱等值）。[[68]](https://osome.com/hk/blog/how-to-file-employers-return-form-bir56a-ir56b/) 这意味着范围非常广泛，不仅包括：

- 最后的工资、佣金、奖金
    
- 代通知金（Payment in lieu of notice）[[72]](https://www.ird.gov.hk/eng/faq/cer.htm)
    
- 约满酬金、递延花红
    
- 即将支付的费用报销
    
- **任何其他有经济价值的利益**，例如在离职时结算的股票期权收益或归属的股权奖励。
    

#### 补报/变更表单

如果在提交原始IR56G之后，公司又需要向该离港员工支付额外的款项（例如，员工离港后归属的递延奖金），雇主必须提交一份**“附加”（Additional）的IR56G表格**来报告这笔额外收入，并同样对该笔款项履行暂扣义务。[[7]](https://www.ird.gov.hk/eng/pdf/ir56g.pdf)

#### 常见踩坑

- **股权激励在离职后归属/兑现**：这是一个非常常见且高风险的场景。员工离职并离港时，可能仍持有未归属的RSU或未行权的期权。
    
    - **如果在离港前行权/归属**：该收益必须在IR56G中申报，并纳入暂扣范围。[[6]](https://yauandwong.com/resources/tax-clearance-and-ir56g-guide/)
        
    - **如果在离港后行权/归属**：
        
        - 如果是**股票期权 (Stock Option)** 在离港后行权，根据税局指引，其收益应通过**补报IR56B**的方式进行申报，而非IR56G。[[11]](https://www.hkwj-taxlaw.hk/ir56g-form-hong-kong/)
            
        - 如果是**限制性股票单位 (RSU) 或股票奖励 (Share Award)** 在离港后归属，实务上通常建议通过提交**附加IR56G**来申报。[[11]](https://www.hkwj-taxlaw.hk/ir56g-form-hong-kong/)
            
        - 无论哪种方式，雇主都有明确的后续申报义务。
            
- **对“money’s worth”范围理解不清**：忽略了非现金形式但有明确价值的支付，如公司承担的搬家费、子女教育费结余等。
    
- **未能提前一个月申报**：最后一刻才发现员工要离港，导致无法满足法定通知期限，这本身已构成违规。
    

#### 企业动作建议

- **离职面谈清单**：在员工离职面谈或办理离职手续时，必须明确询问其未来去向（是否离港），并作为标准流程记录在案。
    
- **联动股权激励管理**：HR的离职流程必须与股权激励管理系统（或台账）联动。当有员工离港时，应立即冻结其任何未结算款项，并检查是否存在未归属/未行权的股权，明确后续的税务处理和申报责任。
    
- **与员工的清晰沟通**：在启动离港清税程序时，向员工书面解释暂扣款项是法定义务，并提供IR56G副本，指导其尽快联系税务局，以缩短款项冻结时间。
    

### 5. 税务违规与逃税后果

香港《税务条例》（Inland Revenue Ordinance, IRO）为税务违规行为设定了从行政处罚到刑事检控的多层次后果。

|   |   |   |
|---|---|---|
|处罚类型|相关条款|主要后果|
|**行政处罚（附加税）**|**第82A条**|- 针对填报不确报税表、未能在限期内提交报税表、未能通知税务局其应课税等行为。<br>    <br>- 税务局局长可对纳税人（包括雇主和雇员）评定**不超过所少征收税款3倍的附加税**作为罚款。[[50]](https://www.ird.gov.hk/eng/pol/ppo.htm)[[51]](https://sleek.com/hk/resources/hong-kong-tax-compliance/)|
|**刑事责任（检控）**|**第80条**|- 针对无合理辩解而未能遵从条例规定（如提交报税表）。<br>    <br>- 罚款港币10,000元，并可能被加罚相等于少征收税款3倍的罚款。[[50]](https://www.ird.gov.hk/eng/pol/ppo.htm)|
|**刑事责任（蓄意逃税）**|**第82条**|- 针对**蓄意及意图**逃税的行为，如在报税表中漏报任何应填报的款项、作出任何虚假的陈述、提供虚假资料等。<br>    <br>- 一经定罪，可处**罚款港币50,000元**，加罚**相等于少征收税款3倍的罚款**，以及**监禁3年**。[[50]](https://www.ird.gov.hk/eng/pol/ppo.htm)[[51]](https://sleek.com/hk/resources/hong-kong-tax-compliance/)|

**处罚考量因素** 税务局在决定处罚水平时，会综合考虑多种因素，包括：

- **行为的严重性**：是疏忽、鲁莽还是蓄意欺骗？
    
- **持续时间**：是一次性错误还是持续多年的行为？
    
- **涉及金额**：少缴的税款数额大小。
    
- **合作程度**：在被税务局调查时是否主动、全面地披露信息。 根据税务局的罚则政策，对于主动、及时、全面披露的个案，罚款可能会大幅降低；而对于采取蓄意隐瞒、伪造文件等恶劣手段的逃税行为，则会从重处罚，甚至启动刑事检控。[[50]](https://www.ird.gov.hk/eng/pol/ppo.htm)
    

#### 合规要点清单

- 认识到税务违规的严重性，其后果远不止补缴税款。
    
- 区分“不小心犯错”（可处以附加税）和“蓄意逃税”（可导致刑事责任）的界限。
    
- 雇主若未能履行申报义务（如未按时提交IR56系列表格），同样会面临罚款。[[57]](https://www.tricorglobal.com/blog/common-pitfalls-in-employer-tax-reporting)
    

#### 常见误区

- **误区一：补上税就没事了**。错误。即使补缴了税款，税务局仍有权追征附加税和罚款。
    
- **误区二：董事个人无需负责**。不完全正确。虽然公司作为法人实体承担主要税务责任，但在某些情况下（如董事授权或签署了明知不实的报税文件），个人也可能被追责。不过，最新的判例澄清了在公司层面，主要处罚对象是公司本身而非签署报税表的董事个人。[[56]](https://kpmg.com/cn/en/home/insights/2022/08/tax-alert-15-hk-the-highest-court-clarified-company-directors-are-not-liable-for-additional-taxes-under-section-82a-for-incorrect-profits-tax-returns-filed.html)[[58]](https://www.gibsondunn.com/hong-kong-court-of-final-appeal-confirms-that-directors-are-not-liable-for-penalty-tax-for-signing-incorrect-company-tax-return/)
    

#### 企业动作建议

- **建立内部举报和审查机制**：鼓励员工和管理者对潜在的税务不合规行为提出疑虑，并定期进行内部审计。
    
- **寻求专业意见**：在遇到复杂的税务问题或被税务局调查时，应立即咨询税务顾问，制定恰当的应对策略。
    
- **保留合理解释的证据**：对于任何可能引起争议的税务处理，都应完整保留相关的商业决策记录、合同、邮件沟通等，以证明“合理辩解”。
    

### 6. AEOI/CRS的影响与未来趋势

自动交换财务账户资料（Automatic Exchange of Financial Account Information, AEOI），通常被称为“共同申报准则”（Common Reporting Standard, CRS），是打击跨境逃税的国际合作框架。

#### CRS的基本机制

- **信息收集**：香港的金融机构（如银行、证券公司、保险公司、基金等）需要识别并收集由“非香港税务居民”持有的金融账户信息。[[89]](https://www.ird.gov.hk/eng/faq/dta_aeoi.htm)[[97]](https://www.hkbea.com/html/en/bea-mpf-aeoi.html)
    
- **信息申报**：金融机构每年会将其识别出的“非香港税务居民”的账户信息报送给香港税务局。
    
- **信息交换**：香港税务局会将收到的信息，自动交换给该账户持有人所属税务管辖区的税务当局。[[89]](https://www.ird.gov.hk/eng/faq/dta_aeoi.htm)
    
- **交换的信息**：包括姓名、地址、税务居民身份、纳税人识别号（TIN）、出生日期、账户号码、账户年终结余或价值，以及年度内的利息、股息、出售金融资产所得总额等。[[89]](https://www.ird.gov.hk/eng/faq/dta_aeoi.htm)
    

#### 对隐匿海外收入的影响

对于在香港工作、但同时是其他CRS参与国家（如中国内地）税务居民的个人而言，CRS意味着其在香港金融机构持有的账户信息，将被交换回其居民国税务当局。这使得通过在香港设立银行账户以隐匿来源于全球（包括香港）的薪酬或其他收入变得极为困难。

#### 未来趋势：加密资产报告框架 (CARF) 及CRS修订

为应对数字经济带来的新挑战，经济合作与发展组织（OECD）推出了“加密资产报告框架”（Crypto-Asset Reporting Framework, CARF），并对CRS进行了相应修订。

- **背景**：香港政府已于**2025年12月**就本地实施CARF及修订CRS展开公众咨询，并计划于2026年完成立法。[[29]](https://www.ird.gov.hk/eng/tax/dta5.htm)
    
- **核心变化**：
    
    - **CARF**：将要求加密资产服务提供商（如加密货币交易所）收集并报告其客户的加密资产交易信息，交换机制与CRS类似。
        
    - **CRS修订**：扩大了金融资产的定义，将某些数字货币产品明确纳入CRS申报范围，并加强了尽职调查程序。
        

**未来合规趋势** 随着CARF和新版CRS的落地，通过持有或交易加密资产来隐匿收入的渠道将被进一步堵塞。税务机关将能够自动获取个人在全球范围内的传统金融账户信息和加密资产交易信息，实现对个人财富更全面的穿透式监管。

#### 合规要点清单

- 员工作为个人，有义务向金融机构准确声明自己的税务居民身份。
    
- 企业应意识到，员工通过海外（包括香港）账户接收薪酬以规避其居民国税务申报的行为，在CRS/AEOI体系下已极易暴露。
    
- 关注CARF在香港的立法进程，为未来可能出现的加密资产薪酬申报要求做好准备。
    

#### 常见误区

- **误区一：只有香港税务居民的信息才会被交换**。错误。CRS交换的是“非香港税务居民”在香港持有的账户信息。一个在香港工作的内地税务居民，其香港账户信息将被交换回内地。
    
- **误区二：CRS只交换存款信息**。错误。交换的信息范围广泛，包括账户余额、利息、股息、出售资产所得等。
    
- **误区三：用加密货币发工资可以避开CRS**。在CARF实施后，此路径将被封堵。
    

#### 企业动作建议

- **员工教育**：提醒拥有多重税务身份或海外账户的员工，关注CRS对其全球税务合规的影响，建议其咨询专业税务顾问。
    
- **合规支付渠道**：确保所有薪酬支付都通过正规、有记录的银行渠道或合规的支付服务商进行，避免使用可能引起反洗钱和税务合规风险的非透明渠道。
    
- **政策跟踪**：指派合规或财务团队跟踪CARF在香港的立法和实施细则，评估其对公司未来可能采纳的（加密资产）激励计划的影响。

## 专题二：股票期权及虚拟货币激励的税务处理

随着人才竞争加剧，股权和虚拟资产激励已成为吸引和留住核心员工的重要工具。然而，这些非传统薪酬的税务处理在香港有其特定规则，企业和员工作为纳税人必须清晰理解并合规申报。

### 1. 股票期权与股权奖励

根据香港《税务条例》第9条，任何因受雇工作而获得的收益都应被征收薪俸税，这其中明确包括了“股份认购权收益”（share option gain）。[[38]](https://www.hkwj-taxlaw.hk/individual-tax-implications-share-based-benefits-hong-kong/)[[43]](https://taxsummaries.pwc.com/hong-kong-sar/individual/income-determination) 税务局通过其发布的《税务条例释义及执行指引第38号（修订版）》（DIPN 38），详细阐述了对各类雇员股份激励的税务处理原则。[[88]](https://www.gov.hk/en/residents/taxes/salaries/salariestax/chargeable/options.htm)

#### 课税时间点：确认收益的关键时刻

香港税务的核心原则是，在员工通过股权激励**实际获得有价值的利益**时才征税。不同类型的股权激励，其课税时点（taxable event）有所不同。

|   |   |   |
|---|---|---|
|激励类型|**香港税务处理**|**与美国税务处理简要对比**|
|**股票期权 (Stock Option)**|- **授予 (Grant)**：不征税。<br>    <br>- **归属 (Vest)**：不征税。<br>    <br>- **行权 (Exercise)**：**征税**。此为核心课税事件。收益在行权日确认。[[40]](https://eqvista.com/esop/esop-taxation-in-hong-kong/)<br>    <br>- **转让/放弃 (Assign/Release)**：若有偿转让或放弃期权，**征税**。|- **NSO (非合格期权)**：行权时征税，与香港类似。<br>    <br>- **ISO (激励性期权)**：行权时通常不征收常规所得税（但可能触发AMT），在出售股票时才按资本利得征税，规则更复杂。|
|**限制性股票单位 (RSU) / 股票奖励 (Share Award)**|- **授予 (Grant)**：不征税。<br>    <br>- **归属/交付 (Vest/Delivery)**：**征税**。当股票解除限制、实际交付给员工时，确认为收入。[[37]](https://www.lexology.com/library/detail.aspx?g=9c6b86dc-d621-4140-8c25-5c204439f3fa)<br>    <br>- **出售 (Sale)**：不征税（资本利得）。|- **RSU**：归属时征税，与香港类似。在归属时点，雇主通常有强制的预扣义务。|
|**雇员购股计划 (ESPP)**|- 在购买股票时，若购买价格低于市场公允价，其**折扣部分**可能被视为应税收益。具体处理取决于计划条款。|- **Qualified ESPP**：在购买时不征税，在出售股票时才计算并征收所得税和资本利得，享受一定税务优惠。|

**跨境任职的时间分摊原则 (Time Apportionment)** 如果一名员工从股权授予到归属/行权期间，同时在香港和香港以外地区提供服务，其股权激励收益需要按比例分摊计税。分摊的基础通常是**在港工作天数**占总工作天数（从授予日到归属/行权日）的比例。员工有责任保留充分的差旅记录和工作证明来支持其分摊计算。

#### 课税基础与计算

- **股票期权**：应税收益 = (行权日股票的市场公允价值 - 员工支付的行权价) × 行权股数。[[88]](https://www.gov.hk/en/residents/taxes/salaries/salariestax/chargeable/options.htm)
    
- **RSU/股票奖励**：应税收益 = 归属日股票的市场公允价值 × 归属股数。
    

**核心特征**：

- **性质**：在香港，上述收益被视为**薪俸收入**，适用2%-17%的累进税率或15%的标准税率。
    
- **资本利得**：员工在行权或归属后获得股票，未来再以更高价格出售该股票所产生的增值部分，属于**资本利得**（Capital Gain）。香港目前**不征收资本利得税**，因此这部分收益是免税的。
    

#### 与其他法域差异的合规提示

- **双重居民身份**：一个同时被香港和另一国（如美国）认定为税务居民的个人，其同一笔股权激励收益可能面临两国的同时征税。此时需要依据两国间的税收协定来避免或抵免双重征税。
    
- **跨境派遣**：员工在集团内不同国家/地区间调动，可能导致同一笔股权激励的归属/行权发生在不同法域，税务处理变得复杂，需要仔细规划和计算分摊。
    

#### 企业申报义务

雇主必须在相应的IR56系列表格中准确申报员工的股权激励收益：

- **IR56B**：在“雇主或其他人所提供地方的租值、股份认购权及其他利益”部分（第11(j)项），填报员工在当年度因行权或归属而实现的收益总额。[[88]](https://www.gov.hk/en/residents/taxes/salaries/salariestax/chargeable/options.htm)
    
- **IR56F/IR56G**：如果员工在离职或离港时/之前实现了股权收益，也必须在这些表格的相应栏目中申报。[[88]](https://www.gov.hk/en/residents/taxes/salaries/salariestax/chargeable/options.htm)
    
- **附表与补报**：对于在离职后才实现的股权收益（如离港后行权的期权），雇主有义务提交补充的IR56B或附加的IR56G进行申报。[[11]](https://www.hkwj-taxlaw.hk/ir56g-form-hong-kong/)
    
- **母公司代发与费用分摊 (Recharge)**：
    
    - 即便股权由境外母公司授予和发放，只要该收益与员工在香港的受雇工作相关，香港雇主仍有申报义务。
        
    - 如果境外母公司就该股权激励的成本向香港子公司进行了书面协议的费用分摊（Recharge），根据税务局指引，香港子公司在满足一定条件下，可以在员工行权或股份归属的时点，将这部分分摊成本作为费用进行税务扣除。[[37]](https://www.lexology.com/library/detail.aspx?g=9c6b86dc-d621-4140-8c25-5c204439f3fa)[[39]](https://www.ird.gov.hk/eng/faq/sbpt.htm)
        

#### 合规要点清单

- 准确识别各类股权激励的正确课税时点（期权→行权，RSU→归属）。
    
- 以公允市场价值为基础计算应税收益。
    
- 对跨境员工的收益进行合理的时间分摊。
    
- 在IR56系列表格中清晰、准确地申报股权激励收益。
    

#### 常见误区

- **误区一：授予期权时就确认收入**。错误，香港税法下，授予本身不产生税务事件。
    
- **误区二：将行权后卖出股票的增值部分也报了薪俸税**。错误，这是资本利得，在香港免税。
    
- **误区三：员工离职了，后续的股权行权/归属与公司无关**。错误，雇主仍有申报义务，需提交补充报税表。
    

#### 企业动作建议

- **制定清晰的内部指引**：向持有股权激励的员工发放税务处理说明，解释课税时点、计算方法及个人申报责任。
    
- **保留记录**：完整保存授予协议、行权/归属通知、公允价值证明（如上市公司的收盘价截图）等文件。
    
- **与薪酬系统集成**：理想情况下，股权激励管理系统应能与薪酬系统对接，自动计算应税收益，并生成导入IR56B的附表。
    

### 2. 虚拟货币（加密资产）激励

随着虚拟资产的兴起，一些创新型企业开始使用加密货币（如比特币、以太币或公司自己发行的代币）作为员工薪酬或激励的一部分。香港税务局对此也表明了其基本税务处理立场。

#### IRD对加密资产薪酬的定性

根据税务局发布的《税务条例释义及执行指引第39号》（DIPN 39）及相关说明，以加密资产支付的薪酬被视为**雇员因受雇工作而产生的入息**，应按**薪俸税**的规定计征。[[22]](https://kpmg.com/cn/en/home/insights/2020/04/tax-alert-8-hk-taxation-of-digital-assets-in-hk.html)[[26]](https://www.internationaltaxreview.com/article/2a6a60xlxd60v3a15yw3l/taxing-digital-assets-in-hong-kong)

- **计税基础**：应税收入的金额，应按雇员**有权收取或实际收取**（以较早者为准）该加密资产时的**公允市场价值**（以港币计算）来确定。[[21]](https://hongkong.monx.team/blog/crypto-taxes-in-hong-kong/)[[22]](https://kpmg.com/cn/en/home/insights/2020/04/tax-alert-8-hk-taxation-of-digital-assets-in-hk.html)
    

#### 稳定币与波动性代币的税务处理

在税务口径上，税务局并未对稳定币（如USDT, USDC）和高波动性代币（如BTC, ETH）作出区分。两者均被视为“虚拟商品”（Virtual Commodity）而非货币。[[24]](https://aplus.hkicpa.org.hk/ird-issues-guidance-on-cryptocurrency-taxation/) 因此，税务处理原则是统一的：

- **核心原则**：均以**取得或应计当日**的港币公允价值入账，作为员工的应税收入。
    
- **估值挑战**：对于高波动性代币，确定“当日公允价值”可能更具挑战性，需要可靠、一致的估值方法。
    

**加密资产薪酬计税与留痕要点清单**

1. **确定归属时点 (Accrual/Receipt Date)**: 明确员工在何时法律上有权获得该加密资产，或何时实际收到该资产。
    
2. **确定公允价值 (Fair Market Value)**:
    
    1. **汇率来源**：选择一个或多个主流、公认的加密货币交易所（如Coinbase, Kraken）在归属时点的港币交易价格作为参考。
        
    2. **时间点**：精确到“日”，甚至“小时”或“分钟”，并保持一致性。例如，统一采用归属当日UTC时间下午4点的收盘价。
        
    3. **记录留痕**：截屏或存档所选交易所的价格页面作为证据。
        
3. **记录钱包地址与交易哈希**:
    
    1. **公司支付钱包**: 记录公司用于支付薪酬的钱包地址。
        
    2. **员工接收钱包**: 要求员工提供其用于接收薪酬的钱包地址。
        
    3. **链上证据**: 保存每笔支付的交易ID/哈希（TxID/Hash），作为支付事实的不可篡改证明。
        
4. **保留雇佣文件**: 雇佣合同或激励计划书中应明确约定以加密资产支付薪酬的条款、数量、归属条件和估值方法。
    

#### 监管框架与合规影响

除了税务，以虚拟资产支付薪酬还需关注香港金融监管框架，特别是《打击洗钱及恐怖分子资金筹集条例》（AMLO）下的虚拟资产服务提供者（VASP）牌照制度。

- **VASP牌照制度**：自2023年6月1日起，任何在香港经营“虚拟资产交易所”业务的主体，都必须向香港证监会（SFC）申领VASP牌照。[[98]](https://www.klgates.com/The-Dual-Licensing-Regime-for-Virtual-Asset-Trading-Platforms-in-Hong-Kong-Part-3-10-5-2023)[[99]](https://www.sidley.com/en/insights/newsupdates/2023/05/hk-vatp-licensing-regime-1-dual-licensing-regime-and-key-personnel-requirements)
    
- **“旅行规则” (Travel Rule)**：持牌VASP在进行虚拟资产转账时，必须收集、记录和传递交易双方（发起方和接收方）的信息，类似于传统银行的电汇要求。[[98]](https://www.klgates.com/The-Dual-Licensing-Regime-for-Virtual-Asset-Trading-Platforms-in-Hong-Kong-Part-3-10-5-2023)
    
- **对企业支付薪酬的影响**：
    
    - **一般不构成经营VASP**：企业仅仅是向自己的员工支付薪酬，通常**不被视为**在“经营虚拟资产交易所”，因此企业自身无需申请VASP牌照。[[107]](https://charltonsquantum.com/wp-content/uploads/2024/12/Hongkong-virtual-assets-regulation.pdf)
        
    - **选择合规渠道**：企业应通过合规的渠道（如持牌交易所或合规的场外交易平台）获取用于支付薪酬的加密资产，并应履行基本的客户尽职调查（KYC）和记录保存义务。
        
    - **避免触及红线**：如果企业不仅向员工发放代币，还提供二级市场交易、代币回购、或为员工间的相互交易提供便利等服务，则可能触及“经营虚拟资产业务”的红线，需要评估持牌风险。
        

#### 企业申报与法律责任

- **信息披露**：与股权激励一样，以加密资产支付的薪酬必须在IR56B等表格中申报其港币等值。
    
- **估值留痕**：企业负有举证责任，需证明其用于计算应税收入的公允价值是合理且有据可循的。
    
- **员工报税指引**：企业应指导员工如何在个人报税时正确申报这部分收入。
    
- **审计重点**：税务局在审计时会重点关注：
    
    - 是否以代币支付替代了部分基本工资但未申报。
        
    - 是否故意滞后入账以适用更有利的汇率。
        
    - 是否通过隐匿的钱包地址或境外平台支付以规避申报。
        
    - 是否虚构或操纵公允价值以低报收入。
        

#### 企业动作建议

- **政策先行**：在采纳加密资产激励前，应制定正式的内部政策，明确估值方法、支付流程、税务处理和合规责任。
    
- **选择可靠的估值源**：书面确定并始终使用1-2个公认的、有历史数据可查的交易所作为估值来源。
    
- **加强记录保存**：对每一笔加密资产薪酬支付，都应将支付金额、估值截图、钱包地址、交易哈希等信息打包存档，以备审计。
    
- **法律与税务咨询**：在设计复杂的代币激励计划（特别是涉及公司自行发行的代币）时，必须寻求专业的法律和税务意见。
    

### 3. 反避税与审计重点

税务局会利用《税务条例》中的一般反避税条款（General Anti-Avoidance Provisions），特别是第61条和第61A条，来挑战那些主要目的为获取税收利益而缺乏商业实质的安排。[[51]](https://sleek.com/hk/resources/hong-kong-tax-compliance/)

|   |   |
|---|---|
|常见规避路径|税务局可能的稽核证据链|
|**低价/无偿转让期权给关联方**|- 审查受让方与员工的关系（是否为亲属、受其控制的信托等）。<br>    <br>- 比较转让价格与当时期权的内在价值或评估价值，看是否存在显失公允。|
|**通过离岸空壳公司代持股权/代发薪酬**|- 穿透公司架构，分析最终受益人。<br>    <br>- 审查资金流水，追踪资金是否最终回流至员工个人账户。<br>    <br>- 引用CRS/AEOI交换的账户信息作为证据。|
|**将薪酬收入包装成“资本增值”**|- 审查雇佣合同和激励计划条款，判断支付的性质是基于服务贡献还是投资行为。<br>    <br>- 分析支付的频率和规律性，经常性、固定性的支付更可能被认定为薪酬。|
|**虚构加密资产公允价值**|- 将企业申报的估值与主流交易所在同一时点的价格进行比对。<br>    <br>- 审查企业选择估值源的合理性和一致性。|

**审计核心证据** 税务局在对股权和加密激励进行审计时，通常会要求企业提供以下核心证据：

- **人力资源文件**：雇佣合同、董事会决议、激励计划条款、授予/归属通知书。
    
- **财务记录**：期权/RSU台账、行权/归属记录、工资单、MPF供款记录。
    
- **银行与资金流水**：公司及关联方的银行账户流水，员工个人账户流水，充值/转账记录。
    
- **链上数据 (针对加密资产)**：支付和接收钱包地址、交易哈希、区块浏览器记录。
    
- **第三方文件**：券商结单、公司年报、估值报告。
    

**企业动作建议：**

- **保持一致性**：确保提交给税务局、MPF受托人、公司注册处等不同机构的文件和数据在核心信息上保持一致。
    
- **文档管理**：建立专门的档案，系统性地保存所有与激励计划相关的文件，确保在税务审计时能迅速、完整地提供。
    
- **模拟审计**：定期进行内部自我审查，站在税务局的角度审视自身的激励计划是否存在明显的反避税特征或文档漏洞。
